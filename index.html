Introduzione del progetto

Il progetto riassume le analisi effettuate su due siti web di informazione noti (rainews - ilfattoquotidiano).

Per il primo sito in esame è stata eseguita un'analisi di tipo test con utenti, al fine da testarne l'usabilità. Per il secondo invece sono state testate le barriere di accessibilità attraverso una simulazione euristica.

Le analisi sono state effettuate da tre studenti del corso di Tecnologie Web e Multimediali dell'Università degli Studi di Udine (Miotto, Rizzato, Tonet) sotto richiesta del prof. G. Branjik durante il corso di Progettazione di Siti e Portali Web.

Test con utenti

Il test di usabilità è un metodo di valutazione di prodotti e servizi digitali basato sul coinvolgimento degli utenti.

E' considerato spesso come lo strumento più efficace per capire se un sito web o un prototipo presenta problemi di usabilità. Osservare "all'opera" i potenziali utenti del prodotto da valutare consente l'individuazione immediata di eventuali mancanze/errori e punti di forza. I dati raccolti in fase di test servono a identificare le difficoltà incontrate dagli utenti, individuarne le cause e porvi rimedio, spesso le soluzioni sono indicate dagli stessi utenti.

Valutazione Conformità Linee Guida WCAG 2.0

Uno dei modi per testare le barriere d'accessibilità di un sito web è quello di valutarne la conformità di alcune caratteristiche.

Le caratteristiche considerate sono le linee guida del WCAG (Web Content Accessibility Guidelines) 2.0. Create per guidare lo sviluppatore alla creazione di uno prodotto accessibile dalla più ampia gamma di utenti. Le linee guida, attualmente alla versione 2.0, sono pubblicate dal WAI (Web Accessibility Initiative) che fa parte del W3C (World Wide Web Consortium).
